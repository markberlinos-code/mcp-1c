# Установка 1С и настройка HTTP-сервиса для MCP

Инструкция для тех, у кого ещё не установлена 1С:Предприятие.

> Если 1С уже установлена - переходите сразу к [быстрому старту](getting-started.md).

## Шаг 1. Скачать и установить 1С

**Учебная версия (бесплатно):**

1. Перейти на https://uc1.1c.ru/uchebnaya-versiya-1s/
2. Зарегистрироваться и скачать дистрибутив
3. Запустить установщик, следовать инструкциям (все настройки по умолчанию)

> Учебная версия полностью функциональна, но работает только с файловыми базами и ограничена одним пользователем. Для наших целей этого достаточно.

## Шаг 2. Создать информационную базу

1. Запустить **1С:Предприятие**
2. Нажать **"Добавить"** -> **"Создание новой информационной базы"**
3. Выбрать шаблон (например, **"Бухгалтерия предприятия (учебная)"**) или создать пустую базу
4. Указать имя базы и путь для хранения файлов
5. Нажать **"Готово"**

## Шаг 3. Установить расширение mcp-1c

```bash
# Windows
mcp-1c --install "C:\путь\к\базе"

# macOS
mcp-1c --install ~/Documents/InfoBase

# Linux
mcp-1c --install /path/to/infobase
```

Программа сама найдет платформу, загрузит расширение и обновит конфигурацию БД.

> Если платформа установлена в нестандартную папку:
> `mcp-1c --install "путь" --platform "/custom/path/to/1cv8"`

> Если в базе настроена аутентификация для Конфигуратора (DESIGNER):
> `mcp-1c --install "путь" --db-user Admin --db-password пароль`

> **Клиент-серверная база (MS SQL, PostgreSQL).** Если база зарегистрирована на сервере 1С, добавьте флаг `--server` и укажите строку подключения в формате `сервер\имя_базы`:
> ```bash
> mcp-1c --install "srv-1c\buh_prod" --server --db-user Admin --db-password pass
> ```

**Вручную (если автоустановка не сработала):**

1. Соберите .cfe из исходников (см. [Сборка расширения](../README.md#сборка-расширения-из-исходников))
2. Откройте базу в **Конфигураторе**
3. Меню **Конфигурация -> Расширения конфигурации**
4. Нажмите **"Добавить из файла"** и выберите собранный `MCP_HTTPService.cfe`
5. Нажмите **F7** (обновить конфигурацию базы данных)

## Шаг 4. Запустить HTTP-сервис 1С

**Windows (коммерческая):**
```cmd
"C:\Program Files\1cv8\8.3.XX.XXXX\bin\1cv8.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

**Windows (учебная):**
```cmd
"C:\Program Files\1cv8t\8.5.XX.XXXX\bin\1cv8t.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

**macOS:**

Платформа 1С [не поддерживает HTTP-сервисы на macOS](https://www.online-ufa.ru/content/articles/limit-1c-for-macos/). Используйте Windows (можно в виртуальной машине, например UTM). Автоустановка расширения (`--install`) работает на macOS, но для запуска HTTP-сервиса нужна Windows.

**Linux:**

Встроенный HTTP-сервер (`/HTTPPort`) недоступен. Используйте публикацию через Apache или автономный сервер ibsrv - см. раздел [Публикация через IIS / Apache](#публикация-через-iis--apache) ниже.

> Замените `XX.XXXX` на вашу версию платформы.

Окно 1С откроется и будет работать как HTTP-сервер. **Не закрывайте его**.

**Проверка:**
```bash
curl http://localhost:8080/hs/mcp/version
```

Ожидаемый ответ: `{"version":"0.4.0"}`

## Шаг 5. Настроить AI-клиент

Дальнейшая настройка описана в [быстром старте](getting-started.md#шаг-4-настроить-ai-клиент).

## Публикация через IIS / Apache

Встроенный HTTP-сервер подходит для разработки. Для постоянной работы:

1. Откройте базу в **Конфигураторе от администратора**
2. **Администрирование -> Публикация на веб-сервере**
3. Укажите имя (например `base`), отметьте `MCPService`
4. Нажмите **Опубликовать**

URL в этом случае: `http://localhost/base/hs/mcp`

## Устранение неполадок

**"1C platform not found" при --install**
Платформа не найдена в стандартных папках. Укажите путь явно через `--platform`.

**"address already in use"**
Порт занят. Используйте другой: `/HTTPPort 9090` и обновите URL в конфиге AI-клиента.

**"404" при обращении к /hs/mcp/...**
- Расширение не установлено или не активно. Проверьте: Конфигуратор -> Расширения -> галочка "Активно" напротив MCP_HTTPService.
- Конфигурация БД не обновлена. Нажмите F7 в Конфигураторе.

**"connection refused"**
1С не запущена или запущена без `/HTTPPort`.
