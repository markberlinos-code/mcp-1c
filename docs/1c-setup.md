# Установка 1С и настройка HTTP-сервиса для MCP

Пошаговая инструкция по установке учебной версии 1С:Предприятие и развёртыванию расширения с HTTP-сервисом, к которому подключается MCP-сервер.

## Требования

- Windows 10/11 (учебная версия 1С работает только на Windows)
- ~2 ГБ свободного места на диске
- Права администратора для установки

## Шаг 1. Скачать учебную версию 1С

1. Перейти на https://uc1.1c.ru/uchebnaya-versiya-1s/
2. Зарегистрироваться (бесплатно) и скачать дистрибутив **1С:Предприятие 8.3 (учебная версия)**
3. Запустить установщик, следовать инструкциям (все настройки по умолчанию)

> Учебная версия полностью функциональна, но работает только с файловыми базами и ограничена одним пользователем. Для наших целей этого достаточно.

## Шаг 2. Создать информационную базу

1. Запустить **1С:Предприятие**
2. Нажать **"Добавить"** → **"Создание новой информационной базы"**
3. Выбрать шаблон **"Бухгалтерия предприятия (учебная)"**
4. Указать имя базы (например, `МояБухгалтерия`) и путь для хранения файлов
5. Нажать **"Готово"**

После создания откроется рабочее окно 1С:Предприятия — закройте его, нам нужен **Конфигуратор**.

## Шаг 3. Создать расширение конфигурации

Расширение позволяет добавить HTTP-сервис без изменения основной конфигурации.

1. Запустить базу в режиме **Конфигуратор** (кнопка "Конфигуратор" в списке баз)
2. Меню: **Конфигурация → Расширения конфигурации**
3. Нажать **"Добавить"**:
   - Имя: `MCP_HTTPService`
   - Префикс: `MCP_`
   - Назначение: "Дополнение" (по умолчанию)
4. Нажать **OK**

Расширение появится в списке. Дважды щёлкните по нему, чтобы открыть дерево конфигурации расширения.

## Шаг 4. Добавить HTTP-сервис

### 4.1. Создать HTTP-сервис

1. В дереве расширения: **Общие → HTTP-сервисы** → правый клик → **Добавить**
2. Свойства HTTP-сервиса:
   - Имя: `MCPService`
   - Корневой URL: `mcp`

### 4.2. Добавить шаблон `/metadata`

1. Правый клик на HTTP-сервисе → **Добавить → Шаблон URL**
2. Имя: `Метаданные`, Шаблон: `/metadata`
3. Правый клик на шаблоне → **Добавить → Метод**
4. Имя: `Получить`, HTTP-метод: `GET`
5. Двойной клик на методе — откроется модуль обработчика
6. Вставить код:

```bsl
Функция МетаданныеПолучить(Запрос)
    Результат = Новый Структура;
    Результат.Вставить("Справочники", Новый Массив);
    Результат.Вставить("Документы", Новый Массив);
    Результат.Вставить("Регистры", Новый Массив);

    Для Каждого МетаданныеОбъект Из Метаданные.Справочники Цикл
        Результат.Справочники.Добавить(МетаданныеОбъект.Имя);
    КонецЦикла;

    Для Каждого МетаданныеОбъект Из Метаданные.Документы Цикл
        Результат.Документы.Добавить(МетаданныеОбъект.Имя);
    КонецЦикла;

    Для Каждого МетаданныеОбъект Из Метаданные.РегистрыСведений Цикл
        Результат.Регистры.Добавить(МетаданныеОбъект.Имя);
    КонецЦикла;

    Ответ = Новый HTTPСервисОтвет(200);
    Ответ.УстановитьТелоИзСтроки(
        СтрокаJSON(Результат),
        КодировкаТекста.UTF8,
        ИспользованиеByteOrderMark.НеИспользовать
    );
    Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
    Возврат Ответ;
КонецФункции
```

### 4.3. Добавить шаблон `/object/{type}/{name}`

1. Правый клик на HTTP-сервисе → **Добавить → Шаблон URL**
2. Имя: `Объект`, Шаблон: `/object/{type}/{name}`
3. Правый клик → **Добавить → Метод**
4. Имя: `Получить`, HTTP-метод: `GET`
5. Вставить код:

```bsl
Функция ОбъектПолучить(Запрос)
    ТипОбъекта = Запрос.ПараметрыURL["type"];
    ИмяОбъекта = Запрос.ПараметрыURL["name"];

    МетаданныеОбъекта = Неопределено;

    Если ТипОбъекта = "Catalog" Тогда
        МетаданныеОбъекта = Метаданные.Справочники.Найти(ИмяОбъекта);
    ИначеЕсли ТипОбъекта = "Document" Тогда
        МетаданныеОбъекта = Метаданные.Документы.Найти(ИмяОбъекта);
    ИначеЕсли ТипОбъекта = "Register" Тогда
        МетаданныеОбъекта = Метаданные.РегистрыСведений.Найти(ИмяОбъекта);
    КонецЕсли;

    Если МетаданныеОбъекта = Неопределено Тогда
        Ответ = Новый HTTPСервисОтвет(404);
        Ответ.УстановитьТелоИзСтроки(
            "{""error"": ""Object not found""}",
            КодировкаТекста.UTF8,
            ИспользованиеByteOrderMark.НеИспользовать
        );
        Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
        Возврат Ответ;
    КонецЕсли;

    Результат = Новый Структура;
    Результат.Вставить("Имя", МетаданныеОбъекта.Имя);
    Результат.Вставить("Синоним", МетаданныеОбъекта.Синоним);

    // Реквизиты
    Реквизиты = Новый Массив;
    Для Каждого Реквизит Из МетаданныеОбъекта.Реквизиты Цикл
        ОписаниеРеквизита = Новый Структура;
        ОписаниеРеквизита.Вставить("Имя", Реквизит.Имя);
        ОписаниеРеквизита.Вставить("Синоним", Реквизит.Синоним);
        ОписаниеРеквизита.Вставить("Тип", Строка(Реквизит.Тип));
        Реквизиты.Добавить(ОписаниеРеквизита);
    КонецЦикла;
    Результат.Вставить("Реквизиты", Реквизиты);

    // Табличные части
    ТабличныеЧасти = Новый Массив;
    Для Каждого ТабличнаяЧасть Из МетаданныеОбъекта.ТабличныеЧасти Цикл
        ОписаниеТЧ = Новый Структура;
        ОписаниеТЧ.Вставить("Имя", ТабличнаяЧасть.Имя);

        РеквизитыТЧ = Новый Массив;
        Для Каждого Реквизит Из ТабличнаяЧасть.Реквизиты Цикл
            ОписаниеРеквизита = Новый Структура;
            ОписаниеРеквизита.Вставить("Имя", Реквизит.Имя);
            ОписаниеРеквизита.Вставить("Синоним", Реквизит.Синоним);
            ОписаниеРеквизита.Вставить("Тип", Строка(Реквизит.Тип));
            РеквизитыТЧ.Добавить(ОписаниеРеквизита);
        КонецЦикла;
        ОписаниеТЧ.Вставить("Реквизиты", РеквизитыТЧ);

        ТабличныеЧасти.Добавить(ОписаниеТЧ);
    КонецЦикла;
    Результат.Вставить("ТабличныеЧасти", ТабличныеЧасти);

    Ответ = Новый HTTPСервисОтвет(200);
    Ответ.УстановитьТелоИзСтроки(
        СтрокаJSON(Результат),
        КодировкаТекста.UTF8,
        ИспользованиеByteOrderMark.НеИспользовать
    );
    Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
    Возврат Ответ;
КонецФункции
```

### 4.4. Добавить вспомогательную функцию

Добавьте общий модуль расширения (Общие → Общие модули → Добавить, имя: `MCP_ОбщиеФункции`, галочки: Сервер, Вызов сервера):

```bsl
Функция СтрокаJSON(Значение) Экспорт
    ЗаписьJSON = Новый ЗаписьJSON;
    ЗаписьJSON.УстановитьСтроку();
    ЗаписатьJSON(ЗаписьJSON, Значение);
    Возврат ЗаписьJSON.Закрыть();
КонецФункции
```

> В обработчиках HTTP-сервиса замените вызов `СтрокаJSON(...)` на `MCP_ОбщиеФункции.СтрокаJSON(...)`, если 1С требует явного указания модуля.

## Шаг 5. Обновить расширение и запустить HTTP-сервис

1. В Конфигураторе: **Конфигурация → Обновить конфигурацию базы данных** (F7)
2. Подтвердить изменения

### Вариант A: Встроенный веб-сервер (рекомендуется для разработки)

Запустить 1С:Предприятие с параметром командной строки:

```
"C:\Program Files\1cv8\8.3.xx.xxxx\bin\1cv8.exe" ENTERPRISE /F "C:\путь\к\базе" /HTTPPort 8080
```

Где:
- `8.3.xx.xxxx` — установленная версия платформы
- `/F "C:\путь\к\базе"` — путь к файловой базе
- `/HTTPPort 8080` — порт, на котором будет доступен HTTP-сервис

### Вариант B: Публикация через Apache/IIS

Для production-окружения:

1. В Конфигураторе: **Администрирование → Публикация на веб-сервере**
2. Указать параметры публикации (виртуальный каталог, сервер)

## Шаг 6. Проверить работу

```bash
# Получить дерево метаданных
curl http://localhost:8080/mcp/metadata

# Получить структуру документа
curl http://localhost:8080/mcp/object/Document/РеализацияТоваровУслуг

# Получить структуру справочника
curl http://localhost:8080/mcp/object/Catalog/Контрагенты
```

Ожидаемый ответ `/mcp/metadata`:
```json
{
  "Справочники": ["Контрагенты", "Номенклатура", "Организации", ...],
  "Документы": ["РеализацияТоваровУслуг", "ПоступлениеТоваровУслуг", ...],
  "Регистры": ["КурсыВалют", ...]
}
```

## Шаг 7. Подключить MCP-сервер

После того как HTTP-сервис 1С работает, запустите MCP-сервер:

```bash
./mcp-1c --base http://localhost:8080/mcp
```

Или для Claude Desktop добавьте в `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "1c": {
      "command": "/путь/к/mcp-1c",
      "args": ["--base", "http://localhost:8080/mcp"]
    }
  }
}
```

## Разработка без 1С (mock-сервер)

Если нет доступа к 1С, используйте встроенный mock-сервер для разработки и тестирования:

```bash
# Собрать и запустить mock
go run ./cmd/mock-1c -port 9191

# В другом терминале — запустить MCP-сервер
./mcp-1c --base http://localhost:9191/mcp
```

Mock-сервер эмулирует ответы 1С:
- `/mcp/metadata` — возвращает список справочников, документов, регистров
- `/mcp/object/{type}/{name}` — возвращает структуру для 3 предопределённых объектов:
  - `Document/РеализацияТоваровУслуг`
  - `Catalog/Контрагенты`
  - `Catalog/Номенклатура`

## Устранение неполадок

**Ошибка: "address already in use"**
Порт уже занят. Выберите другой порт: `/HTTPPort 9090` или `-port 9090`.

**Ошибка: "404" при запросе к HTTP-сервису**
- Проверьте, что расширение подключено в 1С (Конфигуратор → Расширения → галочка "Активно")
- Проверьте корневой URL HTTP-сервиса (должен быть `mcp`)
- Убедитесь, что база запущена с параметром `/HTTPPort`

**Ошибка: "connection refused"**
1С не запущена или HTTP-сервис не опубликован. Запустите базу с ключом `/HTTPPort`.

**Кириллица в ответах отображается некорректно**
Убедитесь, что в обработчиках указана кодировка UTF-8: `КодировкаТекста.UTF8`.
